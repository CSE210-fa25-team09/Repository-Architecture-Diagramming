# Security Policy

## Reporting a Vulnerability
If you believe you’ve found a security issue in this project, please contact the maintainers directly.  
Do not open a public GitHub issue, since it may expose sensitive details.

Please include:
- A clear description of the problem  
- Steps to reproduce  
- Any relevant logs or proof of concept  
- Suggested fixes if you have them

We will review your report and respond as quickly as possible.

## Scope
This policy covers all code within this repository, including:
- Diagram generation logic  
- Parsers that read CI/CD configurations  
- API request handling for the diagramming service  

UI styling, documentation, and non-executable assets are considered out of scope.

## Preferred Disclosure
We follow responsible disclosure practices.  
Please allow us time to investigate and patch the issue before any public discussion.

## Dependencies
If the issue originates from an upstream dependency, we may coordinate with the library’s maintainers before responding.

---

Thank you for helping keep the project secure.
